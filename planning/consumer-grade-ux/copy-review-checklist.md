# Consumer UX copy review checklist

Use this checklist before merging any simple-mode UX change.

- [ ] Default help leads with `setup`, `chat`, `status`, `settings`, and `connect-device`.
- [ ] Beginner-facing docs use the mental model: Folder, Safety Level, Connected Devices, Allowed Actions, Activity Log.
- [ ] Simple-mode output does not require understanding `runtimeProfile`, `allowlist`, `subject_hash`, `matcher`, or raw finding IDs.
- [ ] If an internal term must appear, it only appears behind `--advanced`, a raw-details section, or an operator command.
- [ ] Approval copy explains what OR3 wants to do, why it matters, and the risk level.
- [ ] Status output explains problems in user language before showing any technical identifier.
- [ ] Connect-device copy uses role labels such as chat-only, workspace access, or admin device instead of internal role names.
- [ ] Recovery messages include one clear next action or command.
